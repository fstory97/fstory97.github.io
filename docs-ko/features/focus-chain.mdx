포커스 체인은 Cline의 작업 관리 향상 기능으로, 작업 전반에 걸쳐 실시간 진행 상황 추적과 함께 자동 할 일 목록 관리를 제공합니다.

	<img
		src="https://storage.googleapis.com/cline_public_images/docs/assets/2dos.gif"
		alt="실시간 진행 상황 추적을 통한 포커스 체인 할 일 목록 관리"
	/>
</Frame>

이를 통해 Cline은 장기적인 작업을 수행하고, LLM으로 전송되는 컨텍스트를 원활하게 관리하며, 많은 컨텍스트 창 재설정 전반에 걸쳐 Cline을 추적할 수 있습니다.

<Tip>
  포커스 체인은 Cline의 [심층 계획 슬래시 명령](/features/slash-commands/deep-planning)과 특히 잘 작동하여 [계획 프로세스](/features/plan-and-act)를 통해 생성된 구현 작업에 대한 원활한 진행 상황 추적을 제공합니다.
</Tip>

## 주요 기능

### 자동 할 일 목록 생성

Cline은 작업을 분석하고 다음을 포함하는 포괄적인 할 일 목록을 자동으로 생성합니다.
-   마크다운 체크리스트 형식의 명확하고 실행 가능한 항목
-   복잡한 작업을 관리 가능한 단계로 논리적으로 분해
-   작업 진행에 따른 실시간 업데이트

### 사용자 편집 가능 할 일 목록

할 일 목록은 편집 가능한 마크다운 파일로 저장됩니다.
-   선호하는 마크다운 편집기를 통한 직접 편집
-   사용자가 변경한 내용 자동 감지
-   Cline의 워크플로에 원활하게 통합
-   작업 헤더의 편집 버튼을 통해 빠른 액세스

### 시각적 진행 상황 추적

작업 헤더는 명확한 진행 상황 표시기를 표시합니다.
-   현재 진행 상황을 보여주는 **단계 카운터**(예: "3/8")
-   체크 표시로 명확하게 표시된 **완료된 항목**
-   표시기로 강조 표시된 **현재 작업**
-   전체 할 일 목록을 볼 수 있는 **확장 가능한 보기**

### 스마트 알림 시스템

구성 가능한 알림은 할 일 목록이 최신 상태를 유지하도록 보장합니다.
-   6개 메시지마다 기본 알림(1-100개 사용자 지정 가능)
-   계획 모드에서 실행 모드로 전환할 때 자동 프롬프트
-   할 일 목록을 수동으로 편집할 때 사용자 트리거 업데이트

## 시작하기

<Steps>
  <Step title="Cline 설정 열기">
      - Cline 사이드바의 톱니바퀴 아이콘을 클릭합니다.
      - "기능" 섹션으로 이동합니다.
  </Step>
  <Step title="포커스 체인 활성화">
      - "포커스 체인 활성화"를 선택합니다.
      - 선택적으로 "Cline 알림 간격"을 조정합니다(기본값: 6개 메시지).
  </Step>
  <Step title="새 작업 시작">
      - 새 작업을 시작합니다.
      - Cline은 자동으로 할 일 목록을 생성하고 관리하기 시작합니다.
  </Step>
</Steps>

| 설정 | 기본값 | 범위 | 설명 |
|---|---|---|---|
| 포커스 체인 활성화 | 비활성화됨 | 켜기/끄기 | 향상된 작업 진행 상황 추적 활성화 |
| Cline 알림 간격 | 6 | 1-100개 메시지 | Cline이 할 일 목록을 업데이트하는 빈도 |

## 사용 예시

#### 1. 작업 시작

포커스 체인이 활성화된 새 작업을 시작할 때:

```markdown 사용자 요청
사용자: "내 React 앱을 위한 사용자 인증 시스템을 만들어줘"

Cline: [요청 분석 및 할 일 목록 생성]
```

#### 2. 할 일 목록 생성됨

Cline은 작업에 대한 포괄적인 계획을 마크다운 파일로 저장합니다.

```markdown 할 일 목록 생성됨
- [ ] 프로젝트 구조 설정
- [ ] 인증 종속성 설치
- [ ] 사용자 등록 구성 요소 생성
- [ ] 로그인 기능 구현
- [ ] 비밀번호 유효성 검사 추가
- [ ] 사용자 데이터베이스 스키마 설정
- [ ] 인증 테스트 작성
- [ ] 스테이징 환경에 배포
```

#### 3. 진행 상황 추적

Cline이 작업하는 동안 작업 헤더는 실시간 진행 상황을 표시합니다.

```markdown 할 일 목록 헤더
[3/8] 로그인 기능 구현     ⌄
```

클릭하여 전체 목록을 확장합니다.

```markdown 전체 할 일 목록
✓ 프로젝트 구조 설정
✓ 인증 종속성 설치
✓ 사용자 등록 구성 요소 생성
○ 로그인 기능 구현  ← 현재 작업 중
○ 비밀번호 유효성 검사 추가
○ 사용자 데이터베이스 스키마 설정
○ 인증 테스트 작성
○ 스테이징 환경에 배포
```

#### 4. 사용자 편집

할 일 목록을 수정해야 하나요? 문제 없습니다.

<Steps>
  <Step title="할 일 목록 열기">
    확장된 할 일 보기에서 편집 버튼을 클릭합니다.
  </Step>
  <Step title="마크다운 파일 편집">
    편집기에 마크다운 파일이 열립니다.

    ```markdown 할 일 목록 편집
    # 작업 abc123에 대한 포커스 체인 할 일 목록

    <!-- 이 마크다운 파일을 편집하여 포커스 체인 할 일 목록을 업데이트하십시오. -->
    <!-- 미완료 항목에는 - [ ]를, 완료된 항목에는 - [x]를 사용하십시오. -->

    - [x] 프로젝트 구조 설정
    - [x] 인증 종속성 설치 (예: Firebase Auth)
    - [x] 사용자 등록 구성 요소 생성
    - [ ] 로그인 기능 구현
    - [ ] 비밀번호 재설정 기능 추가
    - [ ] 보호된 경로 설정
    - [ ] 로그아웃 기능 구현
    - [ ] 사용자 프로필 페이지 추가
    - [ ] 인증 테스트 작성
    - [ ] 스테이징 환경에 배포

    <!-- 이 파일을 저장하면 작업의 할 일 목록이 업데이트됩니다. -->
    ```
  </Step>
  <Step title="변경 사항 적용">
    필요에 따라 항목을 추가, 제거 또는 재정렬합니다.
  </Step>
  <Step title="파일 저장">
    Cline은 업데이트를 자동으로 감지하고 사용합니다.
  </Step>
</Steps>

## 파일 구조

### 할 일 목록 저장

할 일 목록은 작업 디렉토리에 마크다운 파일로 저장됩니다.

```markdown
<VSCode 전역 저장소>/
  tasks/
    <taskId>/
      focus_chain_taskid_<taskId>.md
      ... 기타 작업 파일
```

### 마크다운 형식

할 일 파일은 표준 마크다운 체크리스트 구문을 사용합니다.

```markdown 예시 할 일 구문
# 작업 abc123에 대한 포커스 체인 할 일 목록

<!-- 이 마크다운 파일을 편집하여 포커스 체인 할 일 목록을 업데이트하십시오. -->
<!-- 미완료 항목에는 - [ ]를, 완료된 항목에는 - [x]를 사용하십시오. -->

- [x] 프로젝트 구조 설정
- [x] 인증 종속성 설치
- [ ] 사용자 등록 구성 요소 생성
- [ ] 로그인 기능 구현
- [ ] 비밀번호 유효성 검사 추가
- [ ] 사용자 데이터베이스 스키마 설정
- [ ] 인증 테스트 작성
- [ ] 스테이징 환경에 배포

<!-- 이 파일을 저장하면 작업의 할 일 목록이 업데이트됩니다. -->
```

## 계획/실행 모드와의 통합

포커스 체인은 Cline의 [계획/실행 모드](/features/plan-and-act)와 원활하게 작동합니다.

-   **계획 모드**: 구체적인 단계를 제시하기 위한 선택적 할 일 목록
-   **실행 모드**: 계획 모드에서 전환할 때 자동 할 일 생성

<Tip>
  복잡한 프로젝트의 경우, 구현을 시작하기 전에 계획 모드에서 접근 방식을 논의하고 다듬으십시오.
</Tip>

## 모범 사례

<AccordionGroup>
  <Accordion title="효과적인 할 일 목록을 위해">
    1. **명확한 요청으로 시작**
      - 상세한 초기 작업 설명 제공
      - 특정 요구 사항 및 제약 조건 포함
      - 선호하는 기술 또는 접근 방식 언급

    2. **생성된 목록 검토**
      - Cline의 분해가 예상과 일치하는지 확인
      - 모든 중요한 단계가 포함되었는지 확인
      - 순서가 프로젝트에 적합한지 확인

    3. **필요할 때 편집**
      - 식별한 누락된 단계 추가
      - 불필요한 항목 제거
      - 더 나은 워크플로를 위해 단계 재정렬
      - 일반 항목에 더 구체적인 세부 정보 추가
  </Accordion>
  <Accordion title="복잡한 프로젝트를 위해">
    1. **계획 모드 먼저 사용**
      - 구현 전에 접근 방식 논의
      - 대화를 통해 요구 사항 다듬기
      - 작업 시작 준비가 되면 실행 모드로 전환

    2. **큰 작업 분해**
      - 복잡한 프로젝트를 더 작고 관리 가능한 작업으로 분할
      - 다른 구성 요소에 대한 별도의 할 일 목록 생성
      - 한 번에 하나의 주요 영역에 집중

    3. **정기적인 검토**
      - 장기 작업 중에 주기적으로 진행 상황 확인
      - 요구 사항이 발전함에 따라 할 일 목록 업데이트
      - 편집을 통해 Cline에 변경 사항 전달
  </Accordion>
  <Accordion title="협업을 위해">
    1. **할 일 파일 공유**
      - 할 일 마크다운 파일은 팀 구성원과 공유할 수 있습니다.
      - 프로젝트 문서화를 위해 버전 제어에 포함
      - 프로젝트 계획 논의의 기초로 사용

    2. **일관된 형식**
      - 표준 마크다운 체크리스트 형식 따르기
      - 항목 설명을 명확하고 실행 가능하게 유지
      - 할 일 목록 전체에서 일관된 용어 사용
  </Accordion>
</AccordionGroup>

## 문제 해결

문제가 있나요? 다음 빠른 수정 사항을 시도해 보십시오.

<AccordionGroup>
  <Accordion title="할 일 목록이 업데이트되지 않나요?">
    - 설정에서 포커스 체인이 활성화되어 있는지 확인
    - 포커스 체인은 작고 기능이 떨어지는 모델에서는 잘 작동하지 않을 수 있습니다.
    - 작업 디렉토리의 파일 권한이 올바른지 확인
  </Accordion>
  <Accordion title="할 일 파일을 편집할 수 없나요?">
    - 편집기가 마크다운을 지원하는지 확인
    - VSCode에 디렉토리에 대한 쓰기 권한이 있는지 확인
  </Accordion>
  <Accordion title="진행 상황이 표시되지 않나요?">
    - 할 일 항목이 올바른 구문(`- [ ]` 및 `- [x]`)을 사용하는지 확인
    - 마크다운 파일이 올바르게 서식 지정되었는지 확인
  </Accordion>
</AccordionGroup>

여전히 문제가 있나요? Cline에서 [/reportbug](/features/slash-commands/report-bug) 명령을 사용하여 도움을 받으십시오.
