# Caret 개발 로드맵

Caret의 각 차별화 기능별 향후 개발 계획과 전략적 방향성을 제시하는 문서입니다.

## 🎯 **전략적 목표**

### **핵심 비전**

- **AI 코딩의 새로운 표준**: Cline을 뛰어넘는 차세대 AI 코딩 어시스턴트
- **글로벌 서비스**: 다국어 지원으로 전 세계 개발자 커뮤니티 확장
- **팀 중심 워크플로우**: 개인 개발자에서 엔터프라이즈까지 확장

### **차별화 전략**

- **완전한 독립성**: Cline 의존성 최소화, 독자적 기술 스택 구축
- **사용자 중심 혁신**: 개발자 커뮤니티 피드백 기반 기능 개발
- **지속가능한 비즈니스 모델**: 구독/조직 기반 수익 모델 확립

---

## 📋 **기능별 개발 로드맵**

### **🔧 Rule Priority System**

#### **단기 계획 (1-2주)**

- **UI 개선**: Rules Toggle Modal에서 우선순위 로직 반영
- **안내 메시지**: 비활성화된 규칙에 대한 사용자 안내
- **로깅 강화**: 어떤 규칙 파일이 선택되었는지 명확한 로그

#### **중기 계획 (1-2개월)**

- **규칙 편집기**: VSCode 내장 규칙 파일 편집 UI
- **템플릿 시스템**: 프로젝트 유형별 규칙 템플릿 제공
- **상속 시스템**: 글로벌/프로젝트 규칙 계층 구조

#### **장기 계획 (3-6개월)**

- **AI 기반 규칙 생성**: 프로젝트 분석 기반 자동 규칙 제안
- **협업 규칙**: 팀 단위 규칙 공유 및 동기화
- **규칙 분석**: 규칙 효과성 측정 및 최적화 제안

#### **장기 계획 (3-6개월)**
