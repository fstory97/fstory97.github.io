## ‚ö° Smart Rule Management

Caret's intelligent rule system prevents configuration conflicts and optimizes token usage:

### Priority System

Rules are loaded in this priority order:

1. `.clinerules` (highest priority)
2. `.cursorrules` (medium priority)
3. `.windsurfrules` (lowest priority)

### Conflict Prevention

- **Single Rule Loading**: Only one rule file is active at a time
- **Automatic Detection**: Caret automatically detects which rule file to use
- **Token Optimization**: Eliminates duplicate instructions across multiple rule files
- **Clear Indication**: UI shows which rule file is currently active

### Benefits

- **No More Confusion**: Never wonder which rules are actually being used
- **Token Efficiency**: Reduced API costs from duplicate instructions
- **Cleaner Configuration**: One source of truth for project rules
