OpenRouter는 단일 API를 통해 다양한 공급자의 광범위한 언어 모델에 액세스할 수 있는 AI 플랫폼입니다. 이를 통해 설정을 단순화하고 다양한 모델을 쉽게 실험할 수 있습니다.

**웹사이트:** [https://openrouter.ai/](https://openrouter.ai/)

### API 키 얻기

1.  **가입/로그인:** [OpenRouter 웹사이트](https://openrouter.ai/)로 이동하세요. Google 또는 GitHub 계정으로 로그인하세요.
2.  **API 키 얻기:** [키 페이지](https://openrouter.ai/keys)로 이동하세요. API 키가 나열되어 있어야 합니다. 그렇지 않은 경우 새 키를 생성하세요.
3.  **키 복사:** API 키를 복사하세요.

### 지원되는 모델

OpenRouter는 방대하고 계속 증가하는 수의 모델을 지원합니다. Cline은 사용 가능한 모델 목록을 자동으로 가져옵니다. 완전하고 최신 목록은 [OpenRouter 모델 페이지](https://openrouter.ai/models)를 참조하세요.

### Cline에서 구성

1.  **Cline 설정 열기:** Cline 패널에서 설정 아이콘(⚙️)을 클릭하세요.
2.  **공급자 선택:** "API 공급자" 드롭다운에서 "OpenRouter"를 선택하세요.
3.  **API 키 입력:** "OpenRouter API 키" 필드에 OpenRouter API 키를 붙여넣으세요.
4.  **모델 선택:** "모델" 드롭다운에서 원하는 모델을 선택하세요.
5.  **(선택 사항) 사용자 지정 기본 URL:** OpenRouter API에 사용자 지정 기본 URL을 사용해야 하는 경우 "사용자 지정 기본 URL 사용"을 선택하고 URL을 입력하세요. 대부분의 사용자는 이 필드를 비워두세요.

### 지원되는 변환

OpenRouter는 모델의 최대 컨텍스트 크기를 초과하는 프롬프트를 처리하는 데 도움이 되는 [선택적 "중간-아웃" 메시지 변환](https://openrouter.ai/docs/features/message-transforms)을 제공합니다. "프롬프트 및 메시지 체인을 컨텍스트 크기로 압축" 확인란을 선택하여 활성화할 수 있습니다.

### 팁 및 참고 사항

-   **모델 선택:** OpenRouter는 다양한 모델을 제공합니다. 필요에 가장 적합한 모델을 찾기 위해 실험해 보세요.
-   **가격:** OpenRouter는 기본 모델의 가격 책정을 기반으로 요금을 청구합니다. 자세한 내용은 [OpenRouter 모델 페이지](https://openrouter.ai/models)를 참조하세요.
-   **프롬프트 캐싱:**
    -   OpenRouter는 캐싱 요청을 지원하는 기본 모델로 전달합니다. 캐싱을 제공하는 모델은 [OpenRouter 모델 페이지](https://openrouter.ai/models)를 확인하세요.
    -   대부분의 모델의 경우 모델 자체에서 지원하는 경우 캐싱이 자동으로 활성화되어야 합니다(Requesty 작동 방식과 유사).
    -   **OpenRouter를 통한 Gemini 모델의 예외:** OpenRouter를 통해 액세스할 때 Google의 캐싱 메커니즘에서 때때로 관찰되는 잠재적인 응답 지연으로 인해 _특히 Gemini 모델의 경우_ 수동 활성화 단계가 필요합니다.
    -   OpenRouter를 통해 **Gemini 모델**을 사용하는 경우 해당 모델의 캐싱을 활성화하려면 공급자 설정에서 "프롬프트 캐싱 활성화" 확인란을 **수동으로 선택**해야 합니다. 이 확인란은 임시 해결책으로 사용됩니다. OpenRouter의 비 Gemini 모델의 경우 이 확인란은 캐싱에 필요하지 않습니다.
