### 개요

모든 사람을 위해 Cline을 더 좋게 만들기 위해, 우리는 개발자들이 오픈 소스 AI 코딩 에이전트를 어떻게 사용하고 있는지 이해하는 데 도움이 되는 사용량 데이터를 수집합니다. 이 피드백 루프는 Cline의 기능과 사용자 경험을 개선하는 데 중요합니다.

우리는 데이터 수집 및 분석을 위해 오픈 소스 분석 플랫폼인 PostHog를 사용합니다. 우리의 원격 측정 구현은 완전히 투명합니다. 우리가 정확히 무엇을 추적하는지 확인하기 위해 [소스 코드](https://github.com/cline/cline/blob/main/src/services/posthog/telemetry/TelemetryService.ts)를 검토할 수 있습니다.

### 추적 정책

개인 정보 보호는 우리의 우선 순위입니다. 기본적으로 수집된 모든 데이터는 익명화됩니다. Cline 계정으로 로그인하면 원격 측정 데이터가 계정과 연결되어 제품을 개선하고 문제가 발생했을 때 더 나은 지원을 제공하는 데 도움이 됩니다. 귀하의 코드, 프롬프트 및 대화 내용은 항상 비공개로 유지되며 절대 수집되지 않습니다.

### 우리가 추적하는 것

우리는 다음을 포함한 기본 사용량 데이터를 수집합니다.

**작업 상호 작용:** 작업 시작 및 종료 시, 대화 흐름(내용 없음)\
**모드 및 도구 사용:** 계획/실행 모드 간 전환, 사용 중인 도구\
**토큰 사용량:** 비용 추정을 위한 대화 길이의 기본 메트릭(토큰의 실제 내용 아님)\
**시스템 컨텍스트:** OS 유형 및 VS Code 환경 세부 정보\
**UI 활동:** 탐색 패턴 및 기능 사용량

완전한 투명성을 위해 우리가 추적하는 정확한 이벤트를 확인하기 위해 [원격 측정 구현](https://github.com/cline/cline/blob/main/src/services/posthog/telemetry/TelemetryService.ts)을 검사할 수 있습니다.

### 옵트아웃 방법

Cline의 원격 측정은 전적으로 선택 사항입니다.

-   VS Code 확장을 업데이트하거나 설치할 때 원격 측정에 대한 메시지가 표시됩니다.
-   설정에서 언제든지 기본 설정을 변경할 수 있습니다.

Cline은 VS Code의 전역 원격 측정 설정도 존중합니다. VS Code 수준에서 원격 측정을 비활성화한 경우 Cline의 원격 측정 또한 자동으로 비활성화됩니다.
