`/reportbug` は Cline で奇妙な問題に遭遇した時の絶対的な救世主です。バグレポートのために GitHub が求めるすべての詳細を覚えておく必要がなく、このコマンドが Cline をあなた専用のバグレポートアシスタントに変えます。

適切なバグレポートに必要なすべての情報の収集をガイドし、適切なフォーマットとシステム詳細を含めて直接 GitHub issues ページに送信します。

#### `/reportbug` スラッシュコマンドの使用

何かおかしくて正常でないと思われることに遭遇した場合：

-   チャットで単に `/reportbug` と入力
-   Cline が必要なすべての詳細をガイドします：
    -   問題を説明する簡潔なタイトル
    -   実際に起こったことと期待していたこと
    -   バグを再現する手順
    -   見た関連する出力やエラー
    -   修正に役立つ可能性のある追加コンテキスト
-   提出前にすべてをレビューできます
-   承認すると、あなたの情報と自動システム詳細を含む完璧にフォーマットされた GitHub issue が開かれます

#### 例

先週、Cline が大きなファイルを読む際にタイムアウトし続ける奇妙なバグに遭遇しました。GitHub テンプレートフィールドをすべて思い出そうとする代わりに、単に `/reportbug` と入力し、Cline がプロセス全体をガイドしてくれました。

何をしようとしていたか、代わりに何が起こったか、そして問題につながった正確な手順について質問されました。最高の部分は、OS バージョン、Cline バージョン、そして開発者が必要とするすべての技術詳細を自動的に含めたことでした。

数秒後、その情報を自分で探し回ることなく、適切にフォーマットされた GitHub issue が作成されました。
