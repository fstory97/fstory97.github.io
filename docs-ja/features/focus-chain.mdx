フォーカスチェーンは、Clineのタスク管理を強化する機能で、タスク全体を通してリアルタイムの進捗追跡と自動的なTodoリスト管理を提供します。

<Frame>
	<img
		src="https://storage.googleapis.com/cline_public_images/docs/assets/2dos.gif"
		alt="Focus Chain todo list management with real-time progress tracking"
	/>
</Frame>

これにより、Clineは長期的なタスクに取り組むことができ、LLMに送信されるコンテキストをシームレスに管理し、多くのコンテキストウィンドウリセットにわたってClineを正しい軌道に保ちます。

<Tip>
  フォーカスチェーンは、Clineの[詳細プランニングスラッシュコマンド](/features/slash-commands/deep-planning)と特に相性が良く、[プランニングプロセス](/features/plan-and-act)を通じて作成された実装タスクのシームレスな進捗追跡を提供します。
</Tip>

## 主要機能

### 自動Todoリスト生成

Clineがあなたのタスクを分析し、包括的なTodoリストを自動的に作成します：
- マークダウンチェックリスト形式による明確で実行可能なアイテム
- 複雑なタスクを管理可能なステップに論理的に分解
- 作業進行に応じたリアルタイムでの更新

### ユーザー編集可能なTodoリスト

Todoリストは編集可能なマークダウンファイルとして保存されます：
- お好みのマークダウンエディタでの直接編集
- あなたが行った変更の自動検出
- Clineのワークフローへのシームレスな統合
- タスクヘッダーの編集ボタンからの素早いアクセス

### ビジュアル進捗追跡

タスクヘッダーに明確な進捗インジケーターを表示：
- 現在の進捗を示す**ステップカウンター**（例：「3/8」）
- チェックマークで明確にマークされた**完了アイテム**
- インジケーターでハイライトされた**現在の作業**
- 完全なTodoリストを確認できる**展開可能なビュー**

### スマートリマインダーシステム

設定可能なリマインダーがTodoリストを常に最新に保ちます：
- デフォルトでは6メッセージごとにリマインド（1-100でカスタマイズ可能）
- プランモードからアクトモードに切り替える際の自動プロンプト
- Todoリストが手動で編集された際のユーザー起動アップデート


## はじめに

<Steps>
  <Step title="Cline設定を開く">
      - Clineサイドバーの歯車アイコンをクリック
      - 「機能」セクションに移動
  </Step>
  <Step title="フォーカスチェーンを有効化">
      - 「フォーカスチェーンを有効にする」をチェック
      - オプションで「Clineリマインド間隔」を調整（デフォルト：6メッセージ）
  </Step>
  <Step title="新しいタスクを開始">
      - 新しいタスクを開始
      - Clineが自動的にTodoリストの作成と管理を開始
  </Step>
</Steps>

| 設定 | デフォルト | 範囲 | 説明 |
|---------|---------|-------|-------------|
| フォーカスチェーンを有効化 | 無効 | オン/オフ | 拡張されたタスク進捗追跡を有効にする |
| Clineリマインド間隔 | 6 | 1-100メッセージ | Clineがtodoリストを更新する頻度 |

## 使用例

#### 1. タスクの開始

フォーカスチェーンを有効にして新しいタスクを開始するとき：

``` markdown ユーザーリクエスト
ユーザー: "Reactアプリのユーザー認証システムを作成してください"

Cline: [リクエストを分析してtodoリストを作成]
```

#### 2. Todoリストの作成

Clineがタスクの包括的なプランを作成し、マークダウンファイルに保存します：

```markdown Todoリスト作成
- [ ] プロジェクト構造のセットアップ
- [ ] 認証依存関係のインストール
- [ ] ユーザー登録コンポーネントの作成
- [ ] ログイン機能の実装
- [ ] パスワード検証の追加
- [ ] ユーザーデータベーススキーマのセットアップ
- [ ] 認証テストの作成
- [ ] ステージング環境へのデプロイ
```

#### 3. 進捗追跡

Clineが作業を進める中で、タスクヘッダーにリアルタイムの進捗を表示：

```markdown Todoリストヘッダー
[3/8] ログイン機能の実装     ⌄
```

クリックして展開し、完全なリストを確認：

```markdown 完全なTodoリスト
✓ プロジェクト構造のセットアップ
✓ 認証依存関係のインストール
✓ ユーザー登録コンポーネントの作成
○ ログイン機能の実装  ← 現在作業中
○ パスワード検証の追加
○ ユーザーデータベーススキーマのセットアップ
○ 認証テストの作成
○ ステージング環境へのデプロイ
```

#### 4. ユーザー編集

Todoリストを調整したい？問題ありません。

<Steps>
  <Step title="Todoリストを開く">
    展開されたtodoビューの編集ボタンをクリック
  </Step>
  <Step title="マークダウンファイルを編集">
    エディタでマークダウンファイルが開きます：

    ```markdown Todoリストの編集
    # Task abc123のフォーカスチェーンTodoリスト

    <!-- このマークダウンファイルを編集してフォーカスチェーンtodoリストを更新 -->
    <!-- 未完了アイテムは - [ ]、完了アイテムは - [x] を使用 -->

    - [x] プロジェクト構造のセットアップ
    - [x] 認証依存関係のインストール（例：Firebase Auth）
    - [x] ユーザー登録コンポーネントの作成
    - [ ] ログイン機能の実装
    - [ ] パスワードリセット機能の追加
    - [ ] 保護されたルートのセットアップ
    - [ ] ログアウト機能の実装
    - [ ] ユーザープロフィールページの追加
    - [ ] 認証テストの作成
    - [ ] ステージング環境へのデプロイ

    <!-- このファイルを保存してタスクのtodoリストを更新 -->
    ```
  </Step>
  <Step title="変更を行う">
    必要に応じてアイテムを追加、削除、または並べ替え
  </Step>
  <Step title="ファイルを保存">
    Clineが自動的に更新を検出して使用
  </Step>
</Steps>

## ファイル構造

### Todoリストの保存

Todoリストはタスクディレクトリにマークダウンファイルとして保存されます：

``` markdown
<VSCode Global Storage>/
  tasks/
    <taskId>/
      focus_chain_taskid_<taskId>.md
      ... その他のタスクファイル
```

### マークダウン形式

Todoファイルは標準的なマークダウンチェックリスト構文を使用します：

```markdown Todoの構文例
# Task abc123のフォーカスチェーンTodoリスト

<!-- このマークダウンファイルを編集してフォーカスチェーンtodoリストを更新 -->
<!-- 形式：未完了アイテムは - [ ]、完了アイテムは - [x] を使用 -->

- [x] プロジェクト構造のセットアップ
- [x] 認証依存関係のインストール
- [ ] ユーザー登録コンポーネントの作成
- [ ] ログイン機能の実装
- [ ] パスワード検証の追加
- [ ] ユーザーデータベーススキーマのセットアップ
- [ ] 認証テストの作成
- [ ] ステージング環境へのデプロイ

<!-- このファイルを保存するとタスクのtodoリストが更新されます -->
```


## プラン/アクトモードとの統合

フォーカスチェーンはClineの[プラン/アクトモード](/features/plan-and-act)とシームレスに動作します：

- **プランモード**: 具体的なステップを提示するためのオプションのTodoリスト
- **アクトモード**: プランモードから切り替える際の自動Todo作成

<Tip>
  複雑なプロジェクトでは、実装のためにアクトモードに切り替える前に、プランモードでアプローチを議論し、洗練させることから始めてください。
</Tip>

## ベストプラクティス

<AccordionGroup>
  <Accordion title="効果的なTodoリストのために">
    1. **明確なリクエストから始める**
      - 詳細な初期タスクの説明を提供
      - 具体的な要件と制約を含める
      - 好みの技術やアプローチがあれば言及

    2. **生成されたリストを確認**
      - Clineの分解があなたの期待と一致するかチェック
      - すべての重要なステップが含まれているか検証
      - プロジェクトにとって順序が意味をなすか確認

    3. **必要に応じて編集**
      - 特定した不足ステップを追加
      - 不要なアイテムを削除
      - より良いワークフローのためにステップを並べ替え
      - 一般的なアイテムにより具体的な詳細を追加
  </Accordion>
  <Accordion title="複雑なプロジェクトのために">
    1. **最初にプランモードを使用**
      - 実装前にアプローチを議論
      - 会話を通じて要件を洗練
      - 作業開始の準備ができたらアクトモードに切り替え

    2. **大きなタスクを分解**
      - 複雑なプロジェクトをより小さく管理可能なタスクに分割
      - 異なるコンポーネントごとに別々のTodoリストを作成
      - 一度に一つの主要エリアに集中

    3. **定期的なレビュー**
      - 長期タスク中は定期的に進捗を確認
      - 要件の進化に応じてTodoリストを更新
      - 編集を通じて変更をClineに伝達
  </Accordion>
  <Accordion title="コラボレーションのために">
    1. **Todoファイルの共有**
      - Todoマークダウンファイルはチームメンバーと共有可能
      - プロジェクトドキュメンテーションのためにバージョン管理に含める
      - プロジェクト計画議論の基礎として使用

    2. **一貫した形式**
      - 標準的なマークダウンチェックリスト形式に従う
      - アイテムの説明を明確で実行可能に保つ
      - Todoリスト間で一貫した用語を使用
  </Accordion>
</AccordionGroup>


## トラブルシューティング

問題が発生していますか？これらの簡単な修正を試してください：

<AccordionGroup>
  <Accordion title="Todoリストが更新されない？">
    - 設定でフォーカスチェーンが有効になっているか確認
    - フォーカスチェーンは小さく能力の低いモデルでは上手く動作しない場合があります
    - タスクディレクトリのファイル権限が正しいか確認
  </Accordion>
  <Accordion title="Todoファイルを編集できない？">
    - エディタがマークダウンをサポートしているか確認
    - VSCodeがディレクトリに書き込み権限を持っているかチェック
  </Accordion>
  <Accordion title="進捗が表示されない？">
    - Todoアイテムが正しい構文（`- [ ]`と`- [x]`）を使用しているか確認
    - マークダウンファイルが適切にフォーマットされているか検証
  </Accordion>
</AccordionGroup>

まだ問題が解決しませんか？Clineの[/reportbug](/features/slash-commands/report-bug)コマンドを使用してヘルプを取得してください。

## 技術詳細（興味のある方向け）

<AccordionGroup>
  <Accordion title="ファイル監視">

    - 完了対総Todoアイテムの動的カウント

  </Accordion>
  <Accordion title="プライバシーに関する考慮事項">
    - TodoリストはVSCodeワークスペースにローカル保存
    - Todoコンテンツは外部サービスに送信されない
    - 使用テレメトリ（設定で無効化可能）
  </Accordion>
</AccordionGroup>
