---
title: "新规则命令"
sidebarTitle: "/newrule"
---

`/newrule` 是一个斜杠命令，让您可以教 Cline 您偏好的工作方式。它在您的 `.clinerules` 目录中创建一个 markdown 文件，作为 Cline 在协助您的项目时应如何行为的持久指令。

可以将其视为设置 Cline 始终遵循的规则，这样您就不必在每次对话中重复您的偏好。

#### 使用 `/newrule` 斜杠命令

当您希望 Cline 始终遵循某些指导原则时：

-   在聊天中输入 `/newrule`
-   Cline 将通过询问您的以下偏好来帮助您创建结构化规则文件：
    -   沟通风格（详细 vs. 简洁）
    -   开发工作流程
    -   编码标准
    -   项目上下文
    -   任何其他特定指导原则
-   您将在创建之前查看规则文件
-   一旦批准，Cline 会在您的 `.clinerules` 目录中创建一个 markdown 文件，该文件将自动加载用于未来的对话

#### 示例

当我厌倦了在每个新任务中重复相同的指令时，我使用了 `/newrule`。我对 React 组件的结构方式、使用哪个测试库，甚至我偏好的变量命名风格都有特定的偏好。

我没有每次都输入这些偏好，而是使用了 `/newrule` 并与 Cline 合作创建了详细的规则文件。我们构建了一个 markdown 文件，涵盖了从代码组织到我对函数式组件而不是类组件的偏好等所有内容。

现在每当我与 Cline 讨论我的 React 项目时，它会自动遵循这些指导原则，而我无需提醒它。最棒的是，我可以为不同的项目创建不同的规则文件，所以 Cline 会适应我正在处理的任何代码库。

#### 灵感

以下是我使用 `/newrule` 使我的开发更顺畅的方式：

-   我为每个主要项目创建了一个规则文件，包含特定的架构模式和库偏好，所以 Cline 总是生成与我们现有代码库匹配的代码。

-   对于我团队的共享项目，我们有一个公共规则文件，确保无论谁使用 Cline 都能保持一致的代码风格和文档实践。

-   在处理遗留代码时，我制作了一个规则文件来提醒 Cline 旧系统的怪癖和约束，所以它从不建议不能很好集成的现代方法。

-   我甚至为我的副项目有一个个人规则文件，包含我所有的固执偏好 - 两空格缩进、到处使用箭头函数，以及我确切的文件夹结构要求。
