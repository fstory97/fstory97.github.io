当您遇到 Cline 的奇怪问题时，`/reportbug` 绝对是救命稻草。您无需记住 GitHub 错误报告所需的所有详细信息，此命令将 Cline 转变为您的个人错误报告助手。

它会引导您收集正确错误报告所需的所有信息，然后直接发送到我们的 GitHub 问题页面，包含所有正确的格式和系统详细信息。

#### 使用 `/reportbug` 斜杠命令

当您遇到看起来不对劲的奇怪问题时：

-   只需在聊天中输入 `/reportbug`
-   Cline 将引导您了解我们需要的所有详细信息：
    -   描述问题的简要标题
    -   实际发生的情况与您期望的情况
    -   重现错误的步骤
    -   您看到的任何相关输出或错误
    -   可能帮助我们修复它的其他上下文
-   您可以在提交之前查看所有内容
-   一旦您批准，它就会打开一个格式完美的 GitHub 问题，包含您的所有信息以及自动系统详细信息

#### 示例

上周我遇到了一个奇怪的错误，Cline 在读取大文件时不断超时。我没有尝试记住所有 GitHub 模板字段，而是直接输入了 `/reportbug`，Cline 引导我完成了整个过程。

它询问我试图做什么，实际发生了什么，以及导致问题的确切步骤。最棒的是，它自动包含了我的操作系统版本、Cline 版本以及我们开发人员需要的所有技术细节。

几秒钟后，我就有了一个格式正确的 GitHub 问题，而无需自己寻找任何这些信息。
